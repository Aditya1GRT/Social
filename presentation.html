Introduction + Abstract

Why this project matters

Social media should help people connect — not manipulate attention or harvest data. The Social Scoop reimagines the experience from the ground up.

📘

Introduction

Social media is a dominant internet communication medium for sharing information, creating content, and building communities. Yet today's major platforms compromise the user experience with ads, algorithmic manipulation, and opaque data policies. The Social Scoop was built to restore simplicity and user trust.

✨

Abstract

A privacy-first social media web app built on the MERN stack — MongoDB, Express, React, and Node.js. Users can register and log in securely, post photos and videos, like and comment, follow people with request approval, chat in real time via Socket.IO, and make voice/video calls with WebRTC. Light/dark themes and a fully responsive layout complete the experience.

Problem Statement

What's broken in social media today

Existing platforms prioritise engagement metrics and advertiser revenue over the actual user experience and their privacy.

📢

Ad-Driven Design

Platforms strategically insert ads into feeds and personalise them using invasive profiling, turning users into the product rather than the customer.

🧠

Algorithmic Feed Manipulation

Non-chronological feeds maximise time-on-platform by surfacing emotionally engaging (often outrage-driven) content rather than posts from people you follow.

🔓

Privacy & Security Gaps

Weak authentication, unencrypted storage, and broad data-sharing practices leave user information exposed — especially on free-tier platforms used by students.

Objectives

What The Social Scoop sets out to achieve

Six clear goals guided every design and development decision throughout the project.

🧼

Clean Minimal UI

Design a clutter-free, responsive interface that works equally well on desktop and mobile without ads or distracting elements.

🚫

No Ads or Data Mining

Build an ad-free product that collects only the minimum data required for the feature to work and stores it securely.

🛡️

Secure Architecture

Protect users with bcrypt password hashing, JWT authentication, and token expiry — so credentials are never stored in plain text.

⚡

Real-Time Communication

Enable live direct messaging, online presence indicators, and WebRTC peer-to-peer video/voice calling through Socket.IO.

📅

Chronological Feed

Show posts in the order they were made — no ranking algorithm, no boosted content, no filtered visibility.

☁️

Durable Media Storage

Persist all photos and videos on Cloudinary CDN so media remains accessible and is automatically cleaned up when posts or accounts are deleted.

Core Features

Everything the platform offers

Eight fully implemented modules that together form a complete social media experience.

🔐

Secure Register & Login

bcrypt-hashed passwords, JWT-signed sessions with 30-day expiry, and automatic redirect to login on token expiry. Signup creates a profile instantly.

🖼️

Post with Photos & Videos

Create posts with text, images, or videos. Media is uploaded to Cloudinary and auto-pauses when scrolled out of view. Like, comment, and delete with optimistic UI.

👥

Follow with Request Approval

Send follow requests that must be approved by the recipient. Followers/following lists, follow-request management, and real-time count updates.

💬

Real-Time Direct Messaging

One-to-one chat powered by Socket.IO with online indicators, photo/video sharing in chat, emoji picker, and smooth mobile layout with back navigation.

📹

WebRTC Voice & Video Calls

Peer-to-peer calls using Google STUN servers and Socket.IO signaling. Accept/reject incoming calls, mute audio, toggle camera, and end calls from both sides.

🔎

User Search & Profiles

Search users by name or username. Profile pages show bio, post grid, followers/following counts, and action buttons (Follow / Unfollow / Message).

🔔

Notifications

In-app notifications for likes, comments, and follow requests. Unread badge on the bell icon, mark-all-read, and profile picture shown per notification.

🌙

Light / Dark Themes

Glassmorphism UI with a full dark/light theme toggle. Preference is persisted per user in the database so it syncs across devices after login.

Technologies Used

MERN stack + supporting tools

A carefully chosen set of modern, production-grade technologies that work together as one cohesive system.

⚛️

React 18

Single-page frontend with hooks, context, and dynamic routing via React Router v6.

🧰

Redux Toolkit

Global state with slices for user, posts, conversations, profile, and notifications. Persisted via redux-persist.

🚀

Node.js + Express

REST API with route-level JWT middleware, multer for file uploads, and cors configuration.

🍃

MongoDB + Mongoose

Atlas M0 cloud database with schema-driven models and a custom adapter for both NeDB (local dev) and MongoDB.

📡

Socket.IO

Bidirectional real-time events for messages, online presence, and WebRTC call signaling (offer/answer/ICE).

📹

WebRTC

Peer-to-peer audio and video calls using RTCPeerConnection with Google STUN servers for NAT traversal.

☁️

Cloudinary

CDN media storage for post photos/videos and profile pictures. Auto-deleted on post or account removal.

🔒

JWT + bcrypt

Stateless authentication with signed tokens (30-day expiry) and bcrypt-hashed passwords at cost factor 10.

Development Methodology

Agile approach to building the app

Work was structured in iterative phases, allowing the team to plan, build, test, and improve continuously.

1

Planning & Requirements

Identified user pain points with existing social platforms. Defined scope: auth, feed, messaging, profiles, follow system, and real-time features. Chose MERN stack based on the team's existing skills and project requirements.

2

Backend API Development

Built Express routes for users, posts, conversations, messages, and notifications. Set up MongoDB Atlas with Mongoose models. Implemented JWT authentication, bcrypt password hashing, and Cloudinary media storage.

3

Frontend & Real-Time Features

Built React pages and components with styled-components (glassmorphism design). Added Redux Toolkit state management, Socket.IO for live chat, WebRTC for video calls, and emoji pickers. Implemented dark/light theme system.

4

Testing, Bug Fixes & Polish

Fixed security holes (unauthenticated API routes), cascade-deleted user data on account removal, cleaned orphaned likes/comments, resolved mobile layout issues, improved post readability, and optimised media loading. Deployed to Render with MongoDB Atlas + Cloudinary.

Future Scope + Novelty

What comes next

The platform has a strong foundation. These are the most meaningful extensions that can be built on top of it.

New Feature

📖 Stories (24-hour Posts)

Disappearing photo/video stories that auto-delete after 24 hours, visible to followers with a ring around the profile picture — similar to Instagram Stories.

New Feature

👥 Group Chats

Multi-person conversations with group name, avatar, admin controls, and the ability to add/remove members. Built on top of the existing Socket.IO messaging system.

Enhancement

🔐 End-to-End Encryption

Encrypt direct messages client-side using the Web Crypto API so that even the server cannot read message content. Keys would be generated and stored per device.

New Feature

📱 Native Mobile App

Rewrite the frontend in React Native to publish on iOS App Store and Google Play with push notifications, native camera access, and offline caching.

New Feature

🔔 Push Notifications

Web Push API integration so users receive like, comment, and follow-request notifications even when the app is closed — without requiring a native app.

Enhancement

🤖 AI Content Moderation

Integrate a lightweight image classification model to automatically flag and reject nudity, violence, or hate-speech content before it is stored in the database.

Research

📊 Creator Analytics

Dashboard for content creators showing post reach, profile visits, follower growth over time, and best-performing content — built entirely on first-party data.

New Feature

🎭 Short-Form Video (Reels)

A dedicated vertical video feed for short clips (15–60 seconds), with looping playback and swipe-up navigation — extending the existing video post infrastructure.

Enhancement

🌐 Internationalisation (i18n)

Add multi-language support using react-i18next so the app can be used in Hindi, Urdu, and other regional languages — making it accessible to a wider student audience.

💡

What Makes This Project Novel

  • ✔ Ad-free and free from algorithmic content ranking
  • ✔ Follow-request approval system for privacy control
  • ✔ Peer-to-peer WebRTC voice & video calling built in
  • ✔ Cascade account deletion cleans all linked data
  • ✔ Full Cloudinary CDN with auto-cleanup on delete
  • ✔ Self-healing DB: orphaned likes/counts fixed on read
  • ✔ Single-binary deploy: frontend + backend on one Render URL
  • ✔ Light/dark theme persisted per user across devices
Conclusion

Practical exposure to full-stack
web development through a
real-world social product

The Social Scoop gave the team hands-on experience with every layer of modern web development — database design, REST API architecture, real-time communication, cloud media storage, frontend state management, and responsive UI. More than a class project, it is a deployable product that solves a genuine problem with principled engineering choices.
